Product Tracking Reactive WebApp (grade: HD)

Created as the capstone project of The Bachelors of IT degree, Leo [full-stack dev] and 5 other group members worked together to develop a product price tracking web app (Runescrape). Throughout the 13-week Runescrapes project, our team adhered to the Agile methodology, dividing tasks and meeting weekly with a lecturer acting as our client. While daily standup communications kept us aligned with our goals

Demonstration of the final product (Runescrape)


In-depth explanation

We adopted a bifurcated architecture, employing ASP .NET and C# for the backend, which enabled us to implement ASP .NET identity for user authentication and Entity Framework for seamless database integration with PostgreSQL.

For website scraping, we employed "browserless" browser service which managed headless chrome instances.

On the frontend, we utilized REACT and tailwind CSS, employing a component-based architecture to expedite development. Hosting was split between Vercel for the frontend and Azure for the backend API. AWS housed our production database.

GitHub served as our collaboration platform, with a 'development' branch for weekly iterations and a 'main' branch for deployment-ready code. Weekly iterations were subject to rigorous unit tests before merging with 'main'.

Live LipSync in Minecraft

A live lip-sync project using Python, OpenCV, and MediaPipe to map facial landmarks in real-time. The system analyzed facial contours, measured distances between key points, and correlated them with nine basic mouth shapes and phonetic symbols. With the MCRCON library, I connected to my local Minecraft server to update my in-game character's model, syncing mouth and eyebrow movements with real-time facial expressions. The system operated at 20 updates per second, enhancing user immersion in the game environment.

Facemesh was selected for its efficiency, crucial for this project's multiplayer functionality. Unlike libraries such as pocketsphinx or other phoneme libraries, which demand seconds of listening time and provide somewhat inaccurate results, Facemesh offered real-time processing and greater accuracy.

Language Lookup-Table Translator/Generator

A multilingual translation utility tailored toward development engine applications that can utilize JSON lookup tables, in it's current state the project prioritized functionality. It leverages diverse resources including the Translate library, Hugging Face models, Azure API, and Docker technology, the system translates text across a spectrum of 120 languages. Unlike conventional solutions like Google Translate, which may lack coverage for specific languages, this tool sidesteps such limitations by integrating various translation methods.

Lookup Table Use Example

Lookup table with translation codes are applicable for localization in such platforms as gaming/software/web environments. Embracing simplicity, the utility doesn't aim to revolutionize language barriers but rather offers a cost-effective hobbbyist approach to facilitating multilingual communication within these frameworks. Notably, it extends its capabilities beyond text translation to include the handling of special font characters through custom Unicode representations that can be specified in a JSON dictionary.
